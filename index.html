Spring 2021, 11 Devs

BeatDown


BeatDown is a rhythm-based singleplayer and multiplayer party game in which players (and AI in the singleplayer mode) control shapes on a board, moving to the beat and crashing into other shapes to destroy them.


My Roles

  • Programmer
  • Game Systems Designer

My Contributions

  • Worked on a rhythm system which can support complex and variable beats
  • Designed and implemented a system for spawning, collecting, and using powerups
  • Implemented quality-of-life features such as control rebinding and user settings

Each player or AI controls a shape that brings a unique Ultimate to the table with which to destroy your enemies.


The chaos wouldn't be complete, however, without the Powerups. I was in charge of implementing the powerups and played a hand in designing them as well. The Potato Bomb was my personal favorite invention: it moves around and can be pushed by players, eventually exploding when its fuse expires and destroying anyone nearby.


In addition to my work on Powerups, the rhythm system, and general bugfixing, I played a part in implementing control rebinding, player preferences such as volume, and the end-of-game screen pictured above.

Slider


Slider is a 2D singleplayer puzzle game that takes inspiration from the classic 8-puzzle: each region is comprised of 9 sliding tiles which must be collected and then arranged to solve the region and progress to the next one.


My Roles

  • Programmer

My Contributions

  • Developed mechanics to support the MagiTech region where players can move between two separate grids of tiles using portals
  • Integrated with the Discord SDK to support Discord Rich Presence Features
  • Assisted in implementing support for control rebinding using the Unity Input System

So far, I have primarily worked on developing mechanics to support the MagiTech region. While the region is currently in development, our end goal is to allow players to travel between two separate grids of puzzle tiles using pairs of portals placed around the region. I was tasked with expanding the existing complex sliding tile system to support this new mechanic.


In addition to working on the MagiTech region, I also tackled implementing Discord Rich Presence features using the Discord Game SDK and assisted in implementing control rebinding using the new(ish) Unity Input System.


Slider was originally a game jam entry, so the build I link below is the original build submitted for the jam. I'll upload a stable, more updated build once our semester work period ends.

Spring 2022, Solo Project

Brussel Bot


With the death of many popular Discord music bots, I decided to take matters into my own hands and create a bot I could count on for myself and my internet friends to use.


Notable Features

  • Uses the DSharpPlus library along with Lavalink to play music over Discord
  • Multi-threaded implementation enables the bot to operate across multiple instances simultaneously
  • Settings defined using an XML file format and parsed with C# XML utilities

The bot includes a suite of commands, including the option to search and select a track from the top results, queue up tracks to play in sequence, and check the status of the currently playing track and queue.


The bot uses C# multithreading to support multiple instances running simultaneously. This isn't necessary for how I use the bot, but I set out to do it for the sake of doing it and I'm quite happy to say that it works well.

DisasterGuard


DisasterGuard is a 3D shooter created in 72 hours for Ludum Dare 50. The game puts the player in the shoes of an employee tasked with holding off the end of the world for their 8 hour shift.


My Roles

  • Programmer
  • Game Systems Designer

My Contributions

  • Designed and implemented key gameplay features including the ability to purchase new weapons
  • Balanced a variety of game mechanics to help achieve a proper difficulty level
  • Designed and implemented overarching code structure to support the variety of disasters in the game

The game focuses on blending gunplay with a variety of disasters for the player to manage. The player must move around the map and interact with objects to keep everything running smoothly.


I especially like the diagetic arcade mini-game for shooting down alien ships before they land. On the gunplay side of things, we had the idea of having the aliens drop candy bars on death which the player could pick up to heal. I suggested having these bars also serve as a currency in addition to healing the player. The healing on kill encourages the player to be agressive and makes it possible to take on multiple aliens with quick shooting and good aim. The currency system also creates a strategic element â€” manning the defense console causes less aliens to appear, which makes the game easier at the cost of giving the player less currency to work with.


We managed to get an impressive amount of content into the game given the amount of time we had and I'm pleased with the way the wide range of mechanics work together to enable various strategic approaches alongside the mechanical challenge of the FPS component.
