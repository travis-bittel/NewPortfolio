Spring 2021, 11 Devs

BeatDown

BeatDown is a rhythm-based singleplayer and multiplayer party game in which players (and AI in the singleplayer mode) control shapes on a board, moving to the beat and crashing into other shapes to destroy them.


My Roles

  • Programmer
  • Game Systems Designer

My Contributions

  • Worked on a rhythm system which can support complex and variable beats
  • Designed and implemented a system for spawning, collecting, and using powerups
  • Implemented quality-of-life features such as control rebinding and user settings



Each player or AI controls a shape that brings a unique Ultimate to the table with which to destroy your enemies.


The chaos wouldn't be complete, however, without the Powerups. I was in charge of implementing the powerups and played a hand in designing them as well. The Potato Bomb (pictured above between the two yellow circles) was my personal favorite invention: it moves around and can be pushed by players, eventually exploding when its fuse expires and destroying anyone nearby.


In addition to my work on Powerups, the rhythm system, and general bugfixing, I played a part in implementing control rebinding, player preferences such as volume, and the end-of-game screen pictured above.

Spring 2022, 15 Devs (Development Ongoing)

Thrall

Thrall is a 2D singleplayer metroidvania with an emphasis on quick movements and limited healing opportunities. The game's signature mechanic is the Healing Dash — after depleting an enemy's health, you can dash through them to execute them, regaining health in the process.


My Roles

  • Programmer
  • Game Systems Designer

My Contributions

  • Created a physics-based movement system featuring 8 unique abilities
  • Designed and implemented a versatile system for supporting moving minecarts with editor-adjustable paths
  • Worked with the Unity Input System to support control rebinding and general input handling



So far, I have worked heavily on player movement and controls, establishing a clean and expandable system for supporting the variety of movement abilities in the game. I also implemented control rebinding using the Unity Input System along with the UI required to support it.


Besides my more general programming tasks, I've also designed and developed a system to support moving minecarts, the signature mechanic for the Mines region. I had the original idea for the mechanic as a way to make the region stand out. While the Mines area is currently under development, we plan for the minecarts to provide a unique puzzle, platforming, and combat experience.


The player can ride along a moving minecart. When jumping off in the same direction it is travelling, the player receives a large boost of momentum, allowing them to cross gaps that would otherwise be impossible.


A stable build will be uploaded here as soon as I have one!

Slider

Slider is a 2D singleplayer puzzle game that takes inspiration from the classic 8-puzzle: each region is comprised of 9 sliding tiles which must be collected and then arranged to solve the region and progress to the next one.


My Roles

  • Programmer

My Contributions

  • Developed mechanics to support the MagiTech region where players can move between two separate grids of tiles using portals
  • Integrated with the Discord SDK to support Discord Rich Presence Features
  • Assisted in implementing support for control rebinding using the Unity Input System



So far, I have primarily worked on developing mechanics to support the MagiTech region. While the region is currently in development, our end goal is to allow players to travel between two separate grids of puzzle tiles using pairs of portals placed around the region. I was tasked with expanding the existing complex sliding tile system to support this new mechanic.


In addition to working on the MagiTech region, I also tackled implementing Discord Rich Presence features using the Discord Game SDK and assisted in implementing control rebinding using the new(ish) Unity Input System.


A stable build will be uploaded here as soon as I have one!



Spring 2021, Solo Project

Spellslinger

Spellslinger is a 2D, sidescrolling game about using spells to defeat enemies made for the GameBoy Advance.


Notable Features

  • Direct Memory Access used to create graphics using sprites and layered backgrounds
  • Hardware interrups and register manipulation used to support layered audio playback
  • Object pooling used to support a variety of enemies



I created this game as my final project for CS 2261 - Media Device Architecture at Georgia Tech. You can tell I'm not much of an artist.


That said, the game makes use of several technical features to support gameplay on the GameBoy Advance. Gameplay features a variety of enemies implemented using an object pooling paradigm along with abilities that are unlocked over time.


The online emulator is a pain, so you'll have to click "Run it anyway" to play.
The controls given in-game are GameBoy Advance buttons, but the defaults for keyboard are:

  • Movement/Levitate: Arrow Keys
  • Shoot: Z
  • Shield: X

Keep in mind that you unlock abilities over time and won't have all of them at the start.
